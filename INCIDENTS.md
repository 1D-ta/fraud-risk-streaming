# Incident Reports (Simulated)

Failure scenarios generated by `scripts/failure_injection/`.

## Incident 1: Label Delay Spike

**Severity:** High

**Trigger:** `make inject-failure SCENARIO=label_delay`

**Symptom:**
- Label maturity rate dropped from 85% to 62%
- P95 delay increased from 6.8 days to 9.2 days
- Training data volume reduced by 23%

**Detection:**
- `artifacts/reports/maturity_analysis.json` shows reduced mature label coverage
- `artifacts/reports/label_delay_stats.json` shows extended delay distribution

**Business Impact:**
- Model retrained on 23% less data
- Precision@500 degraded from 0.78 to 0.71 (estimated)
- Review queue overflow increased 15%

**Mitigation:**
1. Extended maturity window to 10 days (temporary)
2. Reduced review capacity to 425 to maintain precision threshold
3. Monitored label source distribution for upstream issues

**Prevention:**
- Alert: maturity_rate < 0.75 for 2+ consecutive days
- SLA with upstream: P95 delay < 8 days

## Incident 2: Feature Lag

**Severity:** Medium

**Trigger:** `make inject-failure SCENARIO=feature_lag`

**Symptom:**
- Recent rows lose or stale time-window feature values
- Feature freshness degraded for transactions in last 24 hours

**Detection:**
- Feature freshness checks and failure report output
- `artifacts/reports/feature_lag_report.json` flags affected windows

**Business Impact:**
- Lower recall on recent fraud patterns
- Increased false negatives in newest transactions

**Mitigation:**
- Backfill features for affected windows and re-score
- Implement feature freshness monitoring

**Prevention:**
- Alert on feature computation lag > 2 hours
- Monitor feature pipeline execution times

## Incident 3: Distribution Shift

**Severity:** High

**Trigger:** `make inject-failure SCENARIO=distribution_shift`

**Symptom:**
- Amount distribution shifts relative to baseline
- PSI scores exceed threshold in drift detection

**Detection:**
- `monitoring/drift_detection.py` output
- `artifacts/reports/drift_report.json` shows elevated PSI values

**Business Impact:**
- False positives increase
- Queue pressure increases, hitting 500-case capacity limit faster

**Mitigation:**
- Adjust thresholds temporarily
- Retrain on recent representative data
- Review queue prioritization logic

**Prevention:**
- Alert: PSI > 0.25 for any feature
- Weekly model retraining on rolling window

## Incident 4: Label Leakage

**Severity:** Critical

**Trigger:** `make inject-failure SCENARIO=leakage`

**Symptom:**
- Feature logic uses information at or after transaction time
- Temporal validation fails

**Detection:**
- `make check-leakage` fails
- Injection report flags future-information use
- `artifacts/reports/leakage_check.json` shows violations

**Business Impact:**
- Offline metrics become optimistic and unreliable
- Production model performance would degrade significantly
- Trust in evaluation metrics compromised

**Mitigation:**
1. Rebuild features with strict event-time joins
2. Rerun leakage check until clean
3. Retrain all models
4. Re-evaluate on clean feature set

**Prevention:**
- Mandatory leakage check in CI/CD pipeline
- Code review requirement for feature changes
- Automated temporal correctness tests
